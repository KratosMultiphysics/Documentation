Add nodes to ModelPart from an ordered container.

By assuming that the input is ordered (by increasing Id), the nodes can be added more efficiently. Note that the function makes no check of the ordering, it is the responsability of the caller to ensure that it is correct.

Generates entities (Elements or Conditions) based on the provided connectivities and adds them to a specified container.

This function creates entities of type TEntity (Element or Condition) based on their connectivity information. It looks up the specified entities and properties by their IDs in the provided containers and constructs new entities accordingly. The new entities are then added to the specified container. It checks if the specified entity type is registered in Kratos and if the necessary nodes and properties are available. Throws an error if any requirement is not met.

Template Parameters
TEntityThe type of the entities to be generated (Element or Condition). Must be a type registered in Kratos.
Parameters
rEntityNameThe name of the entity type to generate. Must be registered in KratosComponents.
rEntitiesIdsA vector of IDs for the new entities to be created.
rPropertiesIdsA vector of IDs for the properties to be associated with the new entities.
rEntitiesConnectivitiesA vector of vectors, each sub-vector provides the node IDs for the connectivity of one entity.
rThisNodesA reference to the container of nodes available in the current model part.
rThisPropertiesA reference to the container of properties available in the current model part.
rThisEntitiesA reference to the container where the new entities will be added.
Exceptions
std::runtime_errorIf the specified entity type is not registered in Kratos, if any of the specified nodes do not exist, or if any of the specified properties do not exist.

Generates entities (e.g., Elements or Conditions) based on provided connectivities and adds them to a specified container, with an option to use default properties.

This function is designed to create entities of a specific type (denoted by TEntity) based on their node connectivities. It facilitates the construction of these entities by looking up nodes in the provided nodes container and either using a specified Properties object or defaulting to a provided one if not explicitly specified for each entity. The generated entities are then added to the specified container. The function ensures that the entity type is registered in Kratos and that all specified nodes are available. It allows specifying a common Properties object to be used for all entities, which can be particularly useful when all entities share the same physical properties.

Template Parameters
TEntityThe type of entities to be generated (typically Element or Condition). Must be a type registered with KratosComponents.
Parameters
rEntityNameName of the entity type to generate, which must be registered in KratosComponents.
rEntitiesConnectivitiesA vector of vectors, where each sub-vector contains the node IDs defining the connectivity for one entity.
rThisNodesReference to the container holding the nodes available in the current model part.
rThisEntitiesReference to the container where the new entities will be stored.
pPropertiesOptional pointer to a Properties object to be used for all created entities. If not provided (nullptr), the function will use the first Properties object found in rThisProperties for all entities.
Exceptions
std::runtime_errorIf the specified entity type is not registered in Kratos, or if any of the specified nodes do not exist.
